# GB200 (P6e-GB200) on AWS PCS

This reference architecture runs **GB200 NVL72** on [AWS Parallel Computing Service (PCS)](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-parallel-computing-service/) — AWS's managed Slurm service — by adding a P6e-GB200 compute node group to a PCS cluster, with the IMEX wiring the NVLink domain requires.

> **GB200 ≠ B200/B300.** p6e-gb200.36xlarge is a Grace-Blackwell (ARM) node: arm64 DLAMI, 4 Blackwell GPUs/instance, and an 18-instance 72-GPU NVLink domain (NVL36x2). The B200/B300 templates (`add-cng-p6-b200.yaml`, `add-cng-p6-b300.yaml`) are x86 HGX (8-GPU NVLink island, no Grace, no IMEX) — do not mix them.

## How it fits the PCS templates

The base PCS cluster (VPC, cluster, IAM, FSx, login node) is created exactly as for any other family — see the main [README](../README.md) and `assets/pcs-ml-cluster-deploy-all.yaml`. GB200 changes only the **compute node group**:

| Asset | Role |
|---|---|
| `assets/add-cng-p6e-gb200.yaml` | GB200 compute node group: `p6e-gb200.36xlarge`, arm64 (Grace) PCS-Ready DLAMI, EFA NICs, Capacity Block, and IMEX prolog install |
| `assets/scripts/setup-imex.sh` | Installs the IMEX Slurm prolog (forms the NVLink domain per allocation); fetched by the CNG UserData at first boot |

## GB200-specific points

1. **arm64 AMI.** The CNG auto-resolves the **arm64** PCS-Ready DLAMI (`…/dlami-base-ubuntu2404/arm64/latest/ami-id`). Grace is ARM; an x86 AMI will not boot.
2. **IMEX is mandatory.** PCS managed Slurm does not wire IMEX natively. The CNG installs `setup-imex.sh`, which drops a Slurm prolog (`/etc/slurm/prolog.d/91_nvidia_imex.sh`) that builds the IMEX peer list from `$SLURM_JOB_NODELIST` and starts the service, asserting `Domain State: UP`. On Slurm ≥ 24.05 you may instead use `switch/nvidia_imex` in `slurm.conf` and skip the prolog.
3. **No MIG.** CDMM (driver 580) is enabled on GB200 — MIG/vGPU are unsupported.
4. **Capacity Block only.** Set `CapacityReservationId` to your Capacity Block; the CNG launches `MarketType=capacity-block`. Size `MinCount=MaxCount=18` for one full UltraServer (one 72-GPU NVLink domain).
5. **NIC layout is authored-to-spec.** The `NetworkInterfaces` block places one EFA NIC per GPU (cards 1/5/9/13) for the max ~1,600 Gbps; **verify against the instance's actual network-card topology** before a production launch.

## Deploy

```bash
# (base cluster already deployed via pcs-ml-cluster-deploy-all.yaml)
aws cloudformation deploy \
  --template-file assets/add-cng-p6e-gb200.yaml \
  --stack-name pcs-gb200-cng \
  --parameter-overrides \
      ClusterId=pcs_xxxxx ClusterName=my-cluster \
      CapacityReservationId=cr-xxxxxxxx \
      MinCount=18 MaxCount=18 \
      SubnetId=subnet-xxxx ClusterSecurityGroupId=sg-xxxx \
      IamProfileArn=arn:aws:iam::...:instance-profile/... \
      QueueName=gb200 \
      PostInstallScriptUrl=s3://awsome-distributed-ai/templates/aws-pcs/scripts/install-enroot-pyxis.sh \
  --capabilities CAPABILITY_IAM
```

## Validate before running work

```bash
# IMEX / NVLink domain formed across the allocation:
srun -N 18 --ntasks-per-node=1 bash -c 'nvidia-imex-ctl -N | grep -q "Domain State: UP" && echo "$(hostname) UP"'
# Mandatory NCCL correctness gate (see 4.validation_and_observability/6.gb200-health-gate):
sbatch ../../../4.validation_and_observability/6.gb200-health-gate/slurm/health-gate.sbatch
```

## Testability

**Authored-to-spec** until a P6e-GB200 Capacity Block is held. The CNG template, arm64 AMI resolution, IMEX prolog, and validation commands are drawn from AWS/NVIDIA Tier-1 docs and are internally consistent; the EFA NIC card-index map and end-to-end IMEX formation are the items to confirm on a live UltraServer.
